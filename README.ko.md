# OpenSheet

**English docs: [README.md](README.md)**

마이크로소프트 엑셀과 유사하게 동작하는 웹 기반 스프레드시트 편집기입니다.
`.xlsx`와 `.csv` 파일을 열고, 편집하고, **서식을 유지한 채** 다시 저장할 수 있습니다.

브라우저에서 완전히 로컬로 동작하며(파일은 서버로 전송되지 않습니다), 원하면 정적 호스팅으로 웹 배포도 가능합니다.

![main](pics/main.png)

## 실행 방법

**맥 — 터미널 없이:** Finder에서 **`start.command`** 파일을 더블클릭하세요. Node.js
확인 → 최초 1회 의존성 설치 → 서버 실행 → 브라우저 자동 열기까지 알아서 처리합니다.

또는 터미널에서:

```bash
npm install
npm run dev      # http://localhost:5173 자동 실행
```

프로덕션 빌드 / 배포:

```bash
npm run build    # dist/ 에 정적 파일 생성
npm run preview  # 빌드 결과 미리보기
```

| **수식 / 함수** | `=SUM`, `=AVERAGE`, `=IF`, `=VLOOKUP` 등 약 400개 엑셀 호환 함수 ([HyperFormula](https://hyperformula.handsontable.com/) 엔진) |
| **셀 서식** | 굵게 · 기울임 · 밑줄, 글자/채우기 색, 가로·세로 정렬, 텍스트 줄바꿈, 숫자 서식(통화·백분율·소수·날짜), 테두리, 셀 병합 |
| **서식 왕복 보존** | 글꼴, 채우기, 색상(테마·인덱스 색 포함), 숫자 서식, 테두리, 열 너비·행 높이를 `.xlsx`에서 읽고 다시 저장 |
| **조건부 서식** | 규칙(보다 큼/작음·사이·같음·텍스트 포함)으로 셀 강조 — 실시간 평가 및 `.xlsx` 왕복 보존 |
| **셀 메모** | 셀에 메모 첨부(모서리 빨간 표시 + 마우스오버 툴팁), `.xlsx` 주석으로 저장 |
| **행 · 열 편집** | 우클릭 메뉴로 행/열 삽입·삭제 — 수식·병합·메모·규칙이 함께 이동 |
| **찾기 & 바꾸기** | `Ctrl/Cmd+F`로 찾기, `Ctrl/Cmd+H`로 하나씩 또는 모두 바꾸기 |
| **틀 고정** | 앞쪽 행/열을 고정해 스크롤해도 머리글이 보이도록 유지 |
| **자동 채우기** | 채우기 핸들을 드래그해 연속 값(숫자)이나 값/수식 복사 |
| **실행 취소 / 다시 실행** | 셀 내용·서식·테두리·병합·메모·규칙·정렬 전체 이력 관리 |
| **클립보드** | 범위를 TSV로 복사 · 잘라내기 · 붙여넣기 — 앱 내에서는 서식까지 보존, 엑셀/시트와도 호환 |
| **정렬** | 선택 범위를 기준 열로 오름/내림차순 정렬 — 수식의 상대 참조를 이동량만큼 자동 보정 |
| **자동 필터** | 선택 범위에 필터 적용(머리글 행에 드롭다운) — 열별로 값을 골라 해당 행 표시/숨김 |
| **차트** | 선택 범위를 막대 / 선 / 원 차트로 시각화(라벨은 첫 열, 계열 이름은 머리글 행에서) |
| **다중 시트** | 시트 탭 추가 · 삭제 · 이름 변경, 시트 간 참조 |
| **파일 입출력** | `.xlsx` / `.csv` 열기·저장. **원본에 저장**은 열었던 파일에 그대로 덮어씀(크로미엄 계열) 또는 사본 다운로드 |
| **최근 파일** | 최근 연 파일을 빠르게 다시 열기(브라우저에 로컬 저장, 데스크톱·모바일 모두 지원) |
| **자동 저장 & 복구** | 작업 내용을 로컬에 자동 저장하고 다시 열 때 복원 |
| **인쇄 / PDF** | *저장* 메뉴에서 인쇄 또는 PDF 내보내기(그리드 전용 인쇄 레이아웃) |
| **다크 모드** | 시스템 / 밝게 / 어둡게 테마, 상태 표시줄에서 전환 |
| **다국어** | 영어 / 한국어 UI, 상태 표시줄에서 전환(선택 기억) |
| **설치형 앱(PWA)** | 브라우저에서 "설치"하면 독립 창으로 실행되고 **오프라인**에서도 동작(서비스워커가 앱을 캐시). 데스크톱 Chrome/Edge·안드로이드·iOS 지원 |

## 조작법

- **셀 이동**: 방향키 / 클릭 / 드래그(범위 선택), Shift+클릭(범위 확장)
- **편집 시작**: 더블클릭, `Enter`, `F2`, 또는 문자 입력 시작(한글/IME 입력 지원)
- **편집 확정**: `Enter`(아래로) / `Tab`(오른쪽으로), `Esc`(취소)
- **삭제**: `Delete` / `Backspace`
- **실행 취소 / 다시 실행**: `Ctrl/Cmd+Z` / `Ctrl/Cmd+Shift+Z` (또는 `Ctrl+Y`)
- **복사 / 잘라내기 / 붙여넣기**: `Ctrl/Cmd+C` / `X` / `V`
- **찾기 / 바꾸기**: `Ctrl/Cmd+F`(찾기), `Ctrl/Cmd+H`(바꾸기)
- **저장**: `Ctrl/Cmd+S` (파일 선택창으로 연 경우 원본에 저장, 아니면 `.xlsx` 다운로드)
- **인쇄 / PDF**: 툴바 *저장* 메뉴 ▸ *인쇄 / PDF*
- **서식 단축키**: `Ctrl/Cmd+B`(굵게), `Ctrl/Cmd+I`(기울임), `Ctrl/Cmd+U`(밑줄)
- **테두리**: 툴바의 *테두리* 메뉴(모두 / 바깥쪽 / 위 / 아래 / 왼쪽 / 오른쪽 / 없음)
- **조건부 서식**: 툴바 *조건부 서식* 버튼 — 선택 영역에 강조 규칙 추가
- **행·열 삽입·삭제, 셀 메모**: 우클릭(터치는 길게 눌러) 컨텍스트 메뉴
- **자동 채우기**: 선택 영역 오른쪽 아래 모서리의 작은 사각형 드래그
- **틀 고정**: 툴바 *틀 고정* 메뉴
- **수식 입력**: 셀 또는 상단 수식 입력줄에 `=`로 시작하는 식 입력
- **참조 선택**: `=`수식 편집 중 다른 셀을 클릭(또는 드래그)하면 해당 `A1` 참조/범위가 삽입됨
- **열 너비 조절**: 열 머리글 경계 드래그
- **시트 이름 변경**: 시트 탭 더블클릭
- **테마 / 언어**: 상태 표시줄의 선택기에서 전환

하단 상태 표시줄에는 선택 범위의 **개수 · 합계 · 평균**이 실시간 표시됩니다.

## 모바일 앱 (안드로이드 / iOS)

UI가 반응형이라 모바일 브라우저에서 바로 동작합니다. 네이티브 앱 패키징은
[Capacitor](https://capacitorjs.com/)로 구성돼 있고, `android/`·`ios/` 프로젝트가 이미
커밋되어 있어 바로 빌드할 수 있습니다(필요하면 새로 생성도 가능).

**최초 1회 설정** (`android/`·`ios/`가 없을 때만):

```bash
npm install
npx cap add android          # Android Studio 필요
npx cap add ios              # Xcode 필요 (Capacitor 8은 Swift Package Manager 사용 — CocoaPods 불필요)
```

**빌드 & 실행:**

```bash
npm run cap:android          # 웹 빌드 → 동기화 → Android Studio 열기
npm run cap:ios              # → Xcode 열기
```

이후 Run(기기/에뮬레이터) 또는 Archive로 `.apk` / `.ipa`를 만듭니다. 웹 코드를 수정한
뒤에는 `npm run cap:sync`(또는 위 `cap:*` 스크립트가 동기화까지 수행).

**앱 식별자 & 리소스:**

- 앱 ID(안드로이드 `applicationId` / iOS 번들 ID): `com.anttree.opensheet` — 배포 전
  [`capacitor.config.ts`](capacitor.config.ts)(및 네이티브 프로젝트)에서 변경하세요.
- 아이콘·스플래시 원본은 [`assets/`](assets)에 있습니다. 수정 후 모든 크기를 다시 생성하려면:
  ```bash
  npx capacitor-assets generate --ios --android
  ```

> 참고:
> - 안드로이드 빌드는 **JDK 17+** 가 필요합니다. Android Studio가 자체 JDK를 포함하므로
>   (CLI가 구형 시스템 JDK 경고를 내도) Android Studio에서 열면 해결됩니다.
> - **원본에 저장**(File System Access API)은 데스크톱 크로미엄 전용입니다. 모바일에서는
>   열기는 시스템 파일 선택기를, 저장은 사본 다운로드로 동작합니다.

## 기술 스택

- **Vite + React + TypeScript** — UI와 그리드 렌더링
- **HyperFormula** — 엑셀 호환 수식 계산 엔진
- **ExcelJS** — 셀 서식까지 포함한 `.xlsx` 읽기/쓰기
- **Zustand** — 상태 관리

```

  i18n.ts              영어/한국어 문자열 + 언어 스토어
  theme.ts             밝게/어둡게/시스템 테마 스토어
  components/
    Toolbar.tsx        파일 입출력 · 서식 · 테두리 · 병합 · 정렬 · 조건부 서식 버튼
    FormulaBar.tsx     이름 상자 + 수식 입력줄
    Grid.tsx           스프레드시트 그리드(선택·편집/IME·병합·테두리·틀 고정·채우기·메모)
    SheetTabs.tsx      시트 탭
    ContextMenu.tsx    우클릭 / 길게 누르기 메뉴(삽입·삭제·메모·클립보드)
    FindReplace.tsx    찾기 & 바꾸기 패널
    CondFormatPanel.tsx  조건부 서식 규칙 편집기

  icons/               직접 그린 SVG 툴바 아이콘

    fileIO.ts          ExcelJS 기반 xlsx/csv 읽기·쓰기(서식·테두리·크기·메모·규칙)

```

## 알려진 한계

- 그리드는 200행 × 52열(A–AZ)까지 표시합니다(필요 시 `store/useStore.ts`의 `MAX_ROWS`/`MAX_COLS` 조정).
- **원본에 저장**은 File System Access API를 사용하며 크로미엄 계열(Chrome/Edge)에서 동작합니다. 그 외 브라우저에서는 사본 다운로드로 저장됩니다.
- 구형 `.xls` 형식은 지원하지 않습니다 — `.xlsx`로 먼저 저장하세요.
- 테마·인덱스 색상은 기본 Office 팔레트로 해석하므로, 사용자 지정 테마 문서는 색이 약간 다를 수 있습니다.
- 정렬의 수식 참조 보정은 같은 행을 참조하는 일반적인 경우(`=B2*C2`)를 대상으로 합니다.
